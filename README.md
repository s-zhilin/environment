# environment

Набор готовых, переносимых конфигов для самостоятельно хостируемого окружения.
Каждый сервис живёт в своей папке со своим `docker-compose.yml` — можно поднимать
их по отдельности, но все они общаются через одну общую сеть `core`.

## 🚀 Быстрый старт

Все сервисы подключаются к общей внешней сети `core` или `services`. Её нужно создать один раз:

```bash
docker network create core
```

```bash
docker network create services
```

Дальше поднимаем нужный сервис из его папки:

```bash
cd youtrack
docker compose up -d
```

## 🧩 Устройство

- **Один сервис — одна папка.** Внутри лежит `docker-compose.yml` и, при
  необходимости, `.env` с параметрами.
- **Общая сеть `core`.** Объявлена во всех compose как `external`, поэтому
  сервисы видят друг друга по именам контейнеров независимо от того, каким
  проектом они запущены.
- **Данные — в именованных томах.** Стейт переживает пересоздание контейнеров.

Подробности по каждому сервису — в его папке (`docker-compose.yml` и `.env`).

## 🛠 Полезные команды

```bash
docker compose up -d        # поднять сервис (из папки сервиса)
docker compose down         # остановить и удалить контейнеры
docker compose logs -f      # смотреть логи
docker compose pull         # обновить образы
docker compose ps           # статус контейнеров
```
